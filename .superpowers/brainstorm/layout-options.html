Três opções de layout compactado para a Recepção

Mantendo o formulário "Cadastro rápido por exame" inteiro e compactando a lista de atendimentos

OPÇÃO A

Tabela com linhas finas

Lista como mini-tabela: cada atendimento em uma linha com apenas essencial (nome, status, ações). Clique para expandir e ver exames.

│ Nome │ Status │ Exames │ ├─────────────────────────┼─────────────┼────────┤ │ Maria Aparecida (P1) │ Aguardando │ 2/2 │ │ João Silva (Norm) │ Em andamento│ 1/3 │ │ Ana Costa (P2) │ Finalizado │ 3/3 │
Prós
  • Muitos atendimentos visíveis por vez
  • Busca/filtro rápido em grid
  • Layout compacto classicamente entendível
Contras
  • Pode parecer muito "corporativo" para recepção
  • Número de cadastro e notas precisam de expansão
  • Difícil ver prioridade no grid
Clique aqui para selecionar esta opção
OPÇÃO B

Pill/Badge compacto com expansão

Cada atendimento é um "pill" pequeno (altura ~60px) mostrando: Nome, Prioridade (cor), Status badge, progresso dos exames. Clique para expandir detalhes.

┌─ Maria Aparecida (P1) ⚡ │ ⏳ Aguardando │ 2/2 ─┐ └────────────────────────────────────────────────┘ ┌─ João Silva (Normal) 📄 │ ➡️ Em andamento │ 1/3 ┐ └────────────────────────────────────────────────┘
Prós
  • Visual moderno e leve
  • Compacto mas informativo
  • Prioridade clara por ícone/cor
  • Scrollable facilmente
Contras
  • Menos informação por item
  • Pode parecer "minimalista demais"
  • Filtros precisam estar visíveis sempre
Clique aqui para selecionar esta opção
OPÇÃO C

Card "horizontal resumido"

Cards com ~80px de altura. Header: Nome + prioridade em cor. Subtitle: status + progresso. Footer: ícone ⋯ para expandir. Mantém "ar" visual mas ocupa menos espaço.

┏━ Maria Aparecida ⭐ P1 ┣ Aguardando | 2/2 exames ┗ [Expandir ⋯] ┏━ João Silva ┣ Em andamento | 1/3 exames ┗ [Expandir ⋯]
Prós
  • Balanço entre compacto e informativo
  • Mantém visual atual (cards)
  • Fácil de scrollar
  • Muitos itens visíveis ao mesmo tempo
Contras
  • Ainda precisa reduzir a altura original
  • Número de cadastro sai do resumido
Clique aqui para selecionar esta opção

💡 Recomendação: A Opção C mantém a identidade visual do projeto (cards), reduz altura significativamente, e permite expansão de detalhes sem quebrar o fluxo operacional.
